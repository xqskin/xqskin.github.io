适用于maya2018-2020 以2020为例

自动安装 ​

下载get-pip.py

https://bootstrap.pypa.io/pip/2.7/get-pip.py

"C:\Program Files\Autodesk\Maya2020\bin\mayapy.exe" get-pip.py

手动安装 ​

  • 下载安装setuptools = 25.1.3

https://pypi.org/project/setuptools/25.1.3/#files

"C:\Program Files\Autodesk\Maya2020\bin\mayapy.exe" setup.py install

  • 下载pip = 20.3.4

https://pypi.org/project/pip/20.3.4/#files

"C:\Program Files\Autodesk\Maya2020\bin\mayapy.exe" setup.py install

安装 ​

解决使用pip install 时报错（WARNING: Retrying (Retry(total=4, connect=None, read=None, redirect=None, ）

pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

python -m pip install --upgrade pip  -i https://pypi.tuna.tsinghua.edu.cn/simple

安装依赖包

"C:\Program Files\Autodesk\Maya2020\bin\mayapy.exe" -m pip install -r requirements.txt  -i https://pypi.tuna.tsinghua.edu.cn/simple

拓展知识 ​

  • -u 参数 升级现有包
  • -r 参数 使用"-r"参数来指定一个requirements.txt文件
  • python -m pip uninstall pip
  • mayapy.exe -m pip list
  • "C:\Program Files\Autodesk\Maya2020\bin\mayapy.exe" -m pip download requests -d E:\mayagpt
pip download的使用记录 ​
实际开发中，我们的项目部署环境可能是封闭的内网环境，无法直接使用pip install -r requirement.txt这种方式安装项目依赖包，这时pip download这个犀利的工具就要发光发热了
  • 使用pip download在一个可以连接外网的环境下载整个项目的依赖包
pip download \
    --only-binary=:all: \ # 对于包以及包的依赖包，都不使用二进制
    --platform linux_x86_64 \  # 指定系统环境
    -d \home\packs   # 下载的文件的存储目录
    -r requirement.txt    # 指定要下载的包

备注：实际使用中，会发现一些包找不到，比如tornado，会报错如下

ERROR: Could not find a version that satisfies the requirement tornado==6.0 (from -r src/requirements.txt (line 13)) (from versions: none) ERROR: No matching distribution found for tornado==6.0 (from -r src/requirements.txt (line 13)) 原因：没有提供符合条件的二进制包

解决方法：使用非二进制包安装 --no-binary=:all: package_name

pip download --no-binary=:all: tornado==6.0 -d pkg/

  • 使用这些下载好的包

sudo pip3 install -r /xxxx/xxxx/requirement.txt --no-index --find-links /xxxx/xxxxx/xxxxx/（存放下载好的包的目录）

1·之前安装过其他autodesk的软件导致升级了AdskLicensing

2·没有完全卸载干净，或者恢复过系统，有不同版本的AdskLicensing的残余

3·目前的失败的AdskLicensing需要一直联网。

解决办法：

1.卸载高版本AdskLicensingService。在路径【C:\Program Files (x86)\Common Files\Autodesk Shared\AdskLicensing】中找到uninstall.exe程序，双击卸载。

2.安装低版本AdskLicensingService。在Maya的安装包中找到类似如下路径【安装包根目录:\x86\Licensing】的安装文件AdskLicensing-installer.exe，双击安装。

3.重新激活。双击Maya软件，使用激活工具，算出激活码重新激活就可以了。 作者：丿丿丶XTT https://www.bilibili.com/read/cv22457976/ 出处：bilibili
